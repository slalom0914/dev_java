## 로그 설명
```text
ServerSocket Ready...
Socket[addr=/127.0.0.1,port=56606,localport=5000]
```
addr=/127.0.0.1<br>
- 소켓이 접속한 상대방의 IP주소 -> 로컬 호스트(내PC 자신)
- 즉 클라이언트 프로그램이 같은 PC에서 실행 중이라는 뜻임.
- 만일 다른PC에서 접속이면 192.168.0.51처럼 바뀜

port= 56606<br>
- 클라이언트측 포트 번호임
- 클라이언트가 서버에 접속할 때 OS가 자동으로 준 포트 번호임
- 우리가 지정한 5000번은 서버포트(목적지)이고, 클라이언트 본인의
포트 번호는 OS가 자동으로 할당해줌.
- 즉 이 서버는 5000번 포트에서 클라이언트 56606과 연결된 상태라는 의미임

### 정리
TCP연결은 항상 IP + 포트2쌍으로 표현됨
서버는 항상 5000번 포트에서 대기함
접속하는 클라이언트 마다 각기 다른 포트가 부여됨.
그래서 여러 명이 동시에 채팅 서버 5000번에 붙을 수 있는 것임.
-> 이것을 와이어샤크로 보면
- Source: 127.0.0.1, Port: 56606<br>
- Destination: 127.0.0.1, Port: 5000

필터 예:
```text
tcp.port == 5000
```

1. ServerSocket은 문을 열어두는 역할임
- new ServerSocket(5000) -> 5000번 포트 문 열기
- accept()-> 누군가 입장할 때 까지 기다림

2. Socket은 실제로 연결된 통신 통로임
- Socket[addr=/127.0.0.1,port=57195,localport=5000]이것이 통로 정보임
- 한 명 접속할 때마다 Socket객체가 하나씩 생김: 스레드 하나씩 붙여서 채팅 처리
